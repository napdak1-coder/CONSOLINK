# CONSOLINK 프로젝트 계획서 v5.0

> 해운 콘솔사 오퍼레이터를 위한 업무 도우미
> 한국 - 싱가포르 항로 전담 | 2026년 3월

---

## 1. 이 프로그램이 필요한 이유

콘솔사 오퍼레이터는 여러 화주의 소량 화물(LCL)을 모아서 하나의 컨테이너(FCL)로 합쳐 배에 싣는 일을 해요. 이 과정에서 견적, 부킹, 컨테이너 적재, 서류 처리, 마감 관리, 현장 소통을 모두 처리해야 하는데, 현재는 엑셀 + 카카오톡 + 이메일로 따로따로 처리하고 있어요.

### 현재 vs CONSOLINK

| 현재 방식 | 문제점 | CONSOLINK로 해결 |
|-----------|--------|------------------|
| 엑셀로 화물/컨테이너 관리 | 같은 데이터를 여러 파일에 중복 입력 | 한 번 입력하면 모든 화면에 자동 반영 |
| 카톡으로 창고에 지시 | 지시 내용 빠뜨리거나 헷갈림 | 작업 지시서 자동 생성 -> 복사만 하면 끝 |
| 마감 시간을 머릿속으로 기억 | 마감 놓치면 대형 사고 | 카운트다운 + 색상 알람으로 절대 안 놓침 |
| 컨테이너 짤 때 마진 수계산 | 수익성 판단이 늦음 | 화물 추가할 때마다 마진율 실시간 표시 |
| 적재 요구사항 머릿속 정리 | 충돌 나면 사고, 고민 시간 김 | 충돌 자동 감지 + 최적 배치 자동 추천 |

## 3. 핵심 화면 - 적재계획 (메인 작업 공간)

프로그램의 중심 화면이에요. 4개 영역으로 나뉘어요:

| 위치 | 패널명 | 내용 |
|------|--------|------|
| 왼쪽 | 화물 목록 | 미배정 화물 리스트, 드래그해서 컨테이너에 넣기 |
| 가운데 | 3D 컨테이너 | ZONE A(안쪽)/B(중간)/C(문쪽), 화물 색깔 블록 표시 |
| 오른쪽 상단 | 상태 패널 | 적재율 바, 중량 바, 마감 카운트다운, 충돌 상태, 마진 |
| 오른쪽 하단 | 액션 패널 | CFS 지시서 생성, Pre-alert 생성, 배치 확정, 돌발대응 |

**이 화면 하나에서 적재 + 충돌감지 + 마진확인 + 문서생성 + 돌발대응이 다 돼요.**

### 전체 화면 목록

| 화면 | 하는 일 |
|------|---------|
| 로그인 | 이메일 + 비밀번호로 접속 |
| 대시보드 | 마감 카운트다운 + 화물 현황 요약 + 예정 출항 목록 + 체크리스트 |
| 적재계획 | 컨테이너 적재 + 3D 시각화 + 충돌감지 + 마진패널 + CFS지시서생성 |
| 돌발상황 대응 | 화물 지연 등록 + 시나리오 카드 선택 + 재배치 확인 |
| 운임/견적 | R/T 자동판정 + 운임 계산 + 견적서 PDF 출력 + 선사 비교 |
| 화물관리 | 화물 목록 + 상태 관리 + 타임라인 + 필터 |
| 부킹관리 | 선사 부킹 + 마감시간 설정 + 컨테이너 연결 |
| B/L 관리 | MBL/HBL 발행 + Pre-alert 생성 |
| 거래처관리 | 화주 DB + 거래이력 + 신뢰도 + 메모 + 검색 |
| 체크리스트 | 오늘 미완료 항목 + 내일 마감 예정 목록 |
| 템플릿관리 | 카톡/이메일 템플릿 커스터마이징 |
| 설정 | 프로필 + 회사정보 + 항구 DB(KRPUS-SGSIN) + 알림설정 |

## 4. 주요 기능 상세

### 4.1 컨테이너 적재계획 (1순위)

컨테이너 종류:

| 타입 | 최대 부피 | 최대 중량 | 내부 크기 |
|------|-----------|-----------|-----------|
| 20ft | 33.2 CBM | 21,770 kg | 5.9 x 2.35 x 2.39m |
| 40ft | 67.7 CBM | 26,680 kg | 12.03 x 2.35 x 2.39m |
| 40ft HC | 76.3 CBM | 26,460 kg | 12.03 x 2.35 x 2.69m |

ZONE 배치:
- ZONE A (안쪽): 먼저 적재, 나중에 하차하는 화물
- ZONE B (중간): 순서 상관없는 화물
- ZONE C (문쪽): 나중에 적재, 먼저 하차하는 화물

### 4.2 마감시간 관리 (1순위)

| 마감 종류 | 설명 | 놓치면? |
|-----------|------|---------|
| Docu Closing | 서류 마감 - B/L, 인보이스 등 제출 기한 | 선적 불가, 추가 비용 |
| Cargo Cut-off | 창고 반입 마감 - CFS에 화물을 가져와야 하는 기한 | 화물 미선적, 다음 배편 대기 |
| VGM Closing | VGM(총중량) 신고 마감 | 법적 문제, 선적 거부 |

알람 단계:
- 초록 (24시간 이상): 정상
- 노란색 (24~6시간): 경고 + 브라우저 알림
- 빨간색 (6시간 미만): 긴급 알람 + 깜박임 + 사운드
- 회색: 마감 경과

### 4.3 돌발상황 대응 (1순위)

3가지 돌발상황에 대응:

**화물 지연 도착:**
1. "지연" 버튼 클릭 + 예상 도착시간 입력
2. 시나리오 카드 3가지 자동 생성 (기다림/제외/자리비워두기)
3. 각 카드에 마감시간 여유, 마진 변화, 공간 잔량 표시
4. 선택하면 배치도 업데이트 + 화주 알림 메시지 자동 준비

**잘못된 배치 (계획과 다르게 적재):**
- 실제 배치 위치가 계획과 다르면 "배치 불일치 경고" 발생
- "수정 요청" 또는 "현재 배치 승인" 선택 -> 나머지 자동 재조정

**화물 미도착:**
- 마감시간 도래 시 미도착 화물 자동 감지
- 플랜 B 자동 전환 제안

### 4.4 마진 계산

| 구분 | 내용 | 예시 |
|------|------|------|
| 지출 (Buying) | 선사에 내는 FCL 통운임 + THC + 창고비 | $2,500 |
| 수입 (Selling) | 각 화주에게 받는 LCL 운임의 합계 | $3,200 |
| 마진 (Profit) | Selling - Buying = 예상 수익 | $700 (마진율 21.9%) |

마진율 색상: 20% 이상 초록 / 10~20% 노란 / 10% 미만 빨강 / 마이너스 적자 경고

### 4.5 충돌 감지 + 자동 배치

적재 조건 체크항목:
- 적재 위치: 문쪽 배치(먼저 하차) / 안쪽 배치 / 상관없음
- 위치 제약: 바닥 금지 / 상단 금지 / 벽면 배치 / 제약 없음
- 주의사항: 위에 짐 올리지 말 것 / 눌히지 말 것 / 습기 주의 / 충격 주의

충돌 예시:
- A사: "문쪽 배치" + B사: "문쪽 배치" -> 둘 다 문쪽에 못 들어감 -> 충돌!
- A사: "위에 올리지 말것" + B사: "바닥 배치" -> 같은 위치에 걸침 -> 충돌!

## 6. 한계 및 향후 확장

### 초기 버전의 한계
- 마감 알림은 브라우저가 열려있을 때만 작동
- 카카오톡 직접 전송 불가 (텍스트 복사+붙여넣기 방식)
- 3D 시각화는 CBM 비율 기반 근사 표현 (정밀 cm 시뮬레이션 아님)
- 환율은 수동 입력
- 화주 신뢰도는 데이터 최소 30건 이상 쌓여야 정확도 높아짐
- 선박 스케줄 변경은 직접 입력 (선사 시스템 자동 연동은 추후)

### 나중에 추가할 수 있는 기능
- 카카오톡 알림톡 연동
- 이메일 직접 발송
- 선사 운임 테이블 관리
- 실시간 환율 연동
- 통계 대시보드 (월별 물동량, 화주별 매출, 마진 추이)
- 모바일 화면 지원
- 다른 항로 추가 (미국, 유럽, 동남아 등)
