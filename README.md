## 프로젝트 개요

콘솔사 오퍼레이터가 매일 수행하는 컨테이너 적재 계획, 화주 소통, 마감 관리, 문서 생성, 마진 계산 등의 업무를 하나의 웹 프로그램으로 통합하여, 엑셀 + 카카오톡 + 이메일로 분산된 작업 방식을 대체합니다.

**v6.0 업데이트:** 현장 전문가 피드백 반영 + 투명성/감사 추적 시스템 추가

## 작업 규칙 (7대 원칙)

1. **리더 체계**: Project Leader(Opus)가 모든 작업의 최종 의사결정권을 가짐
2. **사전 회의**: 각 작업 시작 전 반드시 팀원들 간 충분한 토론과 설계 회의를 진행
3. **교차 검증**: 모든 코드는 최소 2명 이상의 팀원이 리뷰 후 머지 (Cross-review 필수)
4. **제로 에러**: 오류 없는 코드만 커밋. 테스트 통과 필수
5. **커밋 규칙**: 의미 있는 단위로 커밋, 명확한 커밋 메시지 작성
6. **성능 관리**: 모든 기능은 성능 기준 충족 후 배포
7. **사후 검증**: 작업 완료 후 반드시 실제 동작 확인 (서버 재시작 + 브라우저 검증)

## 메모

- 모든 작업 시작 전 Opus Leader가 전체 설계 리뷰 진행
- 팀원 간 최소 2인 교차 검증(Cross-review) 필수
- 오류 발생 시 즉시 중단하고 근본 원인 분석 후 재작업
- 성능 이슈는 추측으로 해결하지 않고 측정→수정→재측정 루프로 진행
- 실제 브라우저 검증 없이 완료 보고 금지
- v6.0: 현장 전문가 피드백 반영, 투명성/감사 추적 시스템 추가, 부정 방지 5가지 유형 대응
